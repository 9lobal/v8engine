Чек-лист инфраструктуры клиента

Рабочий контур:
-Сервер 1с (без ключа -debug) + SQL server
-Отдельный терминальный сервер для пользователей
-Рабочая база 1с. Режим восстановления - полный (для возможности откатиться на любой момент времени)
-Хранилище конфигурации. Рабочая и тестовая базы подключены к одному хранилищу. Все изменения тестируются на тестовой базе; готовые доработки сохраняются в хранилище; рабочая база обновляется из хранилища

Тестовый контур:
-Тестовый сервер 1с. Работает параллельно рабочему. Ключ -debug. 
-Тестовая база 1с на тестовом сервере 1с. Каждый день разворачивает из рабочей копии. Все доработки тестируются на этой базе
-Консоль администрирования на терминальном сервере для возможности отключить пользователей при обновлении
-MS SQL можно использовать рабочий, предварительно создав несколько баз (пустых)
-Тестовые базы разворачиваются разработчиком при необходимости на ранее созданных базах

Общая информация:
-список пользователей с паролями и контактными данными
-данные о серверах
-данные о размещении баз
